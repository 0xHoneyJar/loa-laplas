## 0 · The one-line frame

The Observatory graduates from grimoire artifact to the **operator's design instrument for Agent-Infra game design** — the surface where gate design, consumption gradients, and agent moves become *visible, rehearsable, and decidable* — built agent-first so any Loa agent can play it, with THE BOX as the eventual customer.

> Sources: Phase 1 (primary why), obs-graduation-brief.md, kickoff-the-box.md:24-28 (attested)

## 1 · Problem Statement

Three problems, one landscape:

1. **Gradients are invisible to their designer.** Gate design, CLI-verb design, and consumption-gradient mechanics are doctrine-in-prose; the operator's stated need: "Visualizing these things in the game UI is the best way for me to understand these things and help me with game design for Agent Infra" (operator, 2026-06-12).
2. **Gates are soft where they claim to be hard.** Live evidence: the compose proof-of-run gate is warn-first prose — "minting a valid_run needs ~6 mechanical steps + 2 env vars; inline role-play costs zero" (playtest-the-territory:35, run ptt-20260611 `valid_run`). The hardness law: "where a PreToolUse HOOK fires → REAL gradient; where it's SKILL.md PROSE → dominant-defect" (environment-design-game-theory:31-34).
3. **Stuck agents have no legal distress move.** Observed: a k-hole agent loitered to termination unable to summon help (operator observation, 2026-06-12, root cause unverified — see R-6). 31% of registry recipes are ghost doors agents can walk into (playtest-the-territory, EXPLORER). The maze has no thread.

> Sources: Phase 1; playtest-the-territory:33-37, 41-52; environment-design-game-theory:25-40

## 4 · Functional Requirements

### FR-D · Graduation FIRST (MVP, sprint 1) — crs `observatory/`
Move engine + producers + CLI + veve to construct-rooms-substrate `observatory/`; Next.js shell (app router) wrapping the sovereign single-file engine. **Embedding spec (flatline BLOCKER-2)**: the engine ships as a SAME-ORIGIN static asset (`public/observatory/game.html`), iframed by the shell route — no cross-origin anywhere in deployment; `file://` remains a LOCAL-dev affordance only, never a deployment mode. **agentation installs in the SHELL** (dev-mode, per its Next.js-only contract). **Sprint-1 shell skeleton defined (flatline IMP-007)**: one route embedding the engine + agentation toolbar + `/api/level` stub; the annotation→/compose improvement loop is wave 2, explicitly. Cross-repo: crs-bound work dispatched via /coord.
> Sources: brief T1; Phase 5 (home=crs); Phase 6 (pulled into MVP, "build in the final home"); agentation SKILL.md (Next.js-only, read 2026-06-12)

### FR-A · The gate vocabulary renders (MVP, with FR-B)
In-game renderings, each measurable against its doctrine source (attested convo 1, background):
- **Receipts-not-rituals**: the gate visibly consumes the ENVELOPE (the receipt) — never an "act" — gateline names the evidence, not the procedure.
- **Three-tier awareness**: room signage plates (tier 1 manifest) · a --help tooltip beat at gates (tier 2) · **the-gate-teaches** (tier 3): refusal gatelines name the exact verb that would pass ("LEGBA_REFUSED: … run `asson harvest` then retry").
- **stderr-as-prompt**: refusal lines carry the corrected invocation.
- **Gate hardness honesty** (Phase 7): hook-backed gates render SOLID; prose-backed gates render HOLLOW. **Mechanism (flatline BLOCKER-1)**: hardness is DECLARED DATA, never inferred at render time — obs-level/1 rev 2 adds `gates[].hardness: hook|prose|unknown`; producers derive it from a per-substrate **hardness manifest** (a maintained mapping: gate → enforcing mechanism, e.g. compose-verify-run = `prose` while warn-first, PreToolUse guards = `hook`); **fallback is fail-honest: `unknown` renders HOLLOW**. The renderer never invents enforcement.

### FR-B · The thread — IMPASSE joins the contract (MVP)
obs-level/1 → **rev 2**: `IMPASSE` joins the verdict enum; `clew` event renders as Ariadne's thread back to the divergence point; routing beats **retrace** (rewind along the thread) / **rotate** (fresh agent, same entry token) / **heal** (a healer construct — GECKO sprite — walks into the room: the dungeon's first summon); the watchdog drops the clew unilaterally at reap (voluntary and involuntary emit the identical packet). **Watchdog ownership (flatline IMP-003)**: the watchdog authority is asson liveness (`@freeside/asson/liveness` → `livenessVerdict`); the compose runtime invokes it and OWNS clew-packet emission — never the distressed agent (testimony only) and never the renderer (consumes attested packets). The three prices render: clew is free-but-ends-the-span; signed into the chain; strictly better than loitering, strictly worse than finishing.
> Sources: attested convo 2 (clew-drop design, background_only); operator k-hole observation; playtest-the-territory Legba-turnstile note

### FR-C · Agents play (MVP-minimal)
The sim grows the action interface: `tick → {read | present | clew}`. **Minimal episode schema (flatline IMP-008)**, JSON-lines, conformance-testable: `{episode_id, tick, actor, action: "read"|"present"|"clew", observation_digest}` — an agent policy plays via `obs` and the renderer replays the episode. Minimal bar this cycle: ONE construct-agent plays one episode end-to-end and its episode is spectated. Golden vectors for the stuck→clew→heal story (G-4). **Policy formalization (flatline IMP-005)**: each named policy (greedy/disciplined/stuck) is a pure decision function `f(quality, clock, params) → action` documented beside sim-gen — divergence claims are testable against the functions, not vibes.
> Sources: Phase 3 (agent-first); brief T3
