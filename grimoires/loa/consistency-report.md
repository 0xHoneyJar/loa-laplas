## Conflicts / Improvement Opportunities

1. **`LEGBA_*` env vars outside the `LOA_*` namespace** (LEGBA_VERIFIED, LEGBA_RECEIPT,
   LEGBA_RUN_DIR, LEGBA_SPAN_INDEX, LEGBA_REEXEC_TOOLS) — defensible as a subsystem
   namespace, but inconsistent with LOA_CLEW_* / LOA_COMPOSE_* precedent.
   Breaking-change class: flag only, do not rename silently.
2. **Two test roots** (tests/integration/ vs tests/composition/state/) — structural
   inconsistency; the second root's path grammar ("composition/state") matches neither
   `integration` nor a suite-per-feature pattern. (See drift D8.)
3. **`.js` vs `.mjs` split** in the same scripts tree — intentional (workflow harness
   files vs ESM CLI) but undocumented; one comment in scripts/lib/ would prevent drift.
4. **`api-routes` concept absent by design** — substrate exposes CLI contracts; exit-code
   vocabularies are consistent (0/1/2/3 dispatch; 0/1/64 cut; 0/1 legba) and documented
   in headers. Keep it that way.

## Verdict

The codebase reads as one mind: header-doc discipline (every script opens with
cycle/PRD/SDD provenance), fail-closed validators, and consistent fixture taxonomy
(valid-*/invalid-*). The only consistency debt is at the seams the repo itself names:
vendored code (clew) and provisional subsystems (legba).
