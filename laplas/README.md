# laplas — the three preparations

**Domain**: toolchain · **Status**: reference v0.1.0 · demo-proven (see poteau/test, 21/21)

The laplas stages the rite: quest (what), party (who), dungeon (where) — prepared
separately, validated together. `bin/laplas-ready.mjs` is the raid lobby: six
cross-checks (P601–P606), every refusal teaches, a pass mints a ready receipt binding
all three manifest hashes. Fixtures in `test/fixtures/` include one doomed module
(collects all six refusals) and one complete one. Poteau (../poteau) is the enforcement
lattice the cooked ceremony runs inside.
